# data/

Clean-room CSV libraries that power the plugin's design and SEO reasoning. When a
skill needs to pick a style, palette, font pairing, or UX rule, it queries one of
these through a Python helper in `scripts/design/`.

> **Originality is a hard rule here.** These files are **generated from first
> principles or from openly-licensed primary sources**, authored for this plugin.
> They are *not* imported, "rewritten," or count-matched from any other plugin.
> Every file records its generation method and any source + license in
> `references/PROVENANCE.md`. Row counts reflect what we actually built, and grow
> organically as the libraries mature.

## Shipping data files

- `ui-styles.csv` — UI style profiles (e.g. minimal, glass, brutalist, soft-UI)
  with our own characteristics / best-fit / anti-pattern columns
- `color-palettes.csv` — palettes generated by our own HSL + contrast script
  (seed and method recorded in PROVENANCE), indexed by industry/mood
- `font-pairings.csv` — heading + body pairings built from open Google Fonts
  metadata with our own pairing rationale
- `product-types.csv` — product/page types mapped to recommended
  pattern/style/color/typography defaults (authored)
- `ux-rules.csv` — UX guidelines in our own words, tagged by priority/category/platform

## Roadmap (not yet shipped)

Planned but not built; nothing references these yet:

- `google-fonts-meta.csv` — Google Fonts metadata (weights, subsets, variable axes)
- `chart-types.csv` — chart types matched to data shapes
- `seo-business-types.csv` — business-profile signals → recommended SEO skill mix
- `industry-templates.csv` — industry overlays for SEO strategy

## How the engine reads these

`scripts/design/design_system.py` loads the relevant CSVs, scores rows against the
user's inputs, and composes a recommendation. The engine is built to tolerate a
**partial** data set: skills work with whatever libraries exist and clearly note
which dimension fell back to heuristics when a library is absent.

## Conventions

- UTF-8, comma-separated, header row first
- One concept per row; `tags` column holds semicolon-separated tags
- Final `source` + `license` columns where a row derives from an external primary
- Final `notes` column may point to deeper docs in `references/`

## Status

Five libraries shipping (clean-room): `ui-styles.csv`, `color-palettes.csv`
(generated, reproducible), `font-pairings.csv`, `product-types.csv`, `ux-rules.csv`.
Per-file method, row counts, and licenses are recorded in `references/PROVENANCE.md`.
